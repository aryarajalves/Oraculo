🧠 O que é o Sistema Oráculo

O Oráculo é uma fábrica autônoma de conteúdo para Instagram, movida por IA, que transforma um tema em um carrossel completo de 10 slides ou em um Reel de vídeo — com copy viral, imagens cinematográficas, narração, trilha e publicação automática.

O que ele produz

FormatoComposiçãoPublicação
Carrossel10 slides (1080×1350px), copy Método Jordânico, imagens IA por slideAutomática via Meta API
Reel de Vídeo7 cenas × (voz + efeito sonoro + vídeo IA + trilha de fundo)Manual (montagem final)

A cadeia de agentes de IA

TEMA (qualquer assunto relevante ao nicho)
    ↓
[Copywriter IA]       → escreve a narrativa com arco emocional (GPT-4o)
    ↓
[Humanizer]           → transforma copy genérica em voz da marca
    ↓
[Revisor/Oráculo]     → pontua e aprova pelo critério viral (5 critérios)
    ↓
[Canalizador Visual]  → decide layout e âncora visual por slide
    ↓
[Diretor Artístico]   → gera imagem por slide via OpenAI gpt-image-2
    ↓
[Compositor]          → monta slide final com texto + imagem
    ↓
[Publisher]           → sobe para ImgBB e publica no Instagram via Meta API

O motor (código) é idêntico para todos os clientes.

🔍 Fase 0 — Pré-Venda

Antes de fechar, verificar se o cliente tem a estrutura mínima para o sistema funcionar. Vender sem isso cria bloqueadores na fase 4.

Perguntas de qualificação

  • Você tem uma conta do Instagram profissional ativa (mín. 90 dias)?
  • Você tem uma Página do Facebook vinculada ao Instagram?
  • Você já tem acesso ao Meta Business Suite?
  • Você tem um produto ou serviço claro que o conteúdo vai promover?
  • Você tem um nicho específico definido?
Se o cliente não tiver a estrutura Meta configurada: incluir no escopo como serviço adicional ou sinalizar como bloqueador antes de fechar.

📋 Fase 1 — Onboarding

Enviar formulário Notion ao cliente. Prazo: 48h para devolver preenchido. Se não preencher corretamente, agendar call de 1h para preencher junto.
Módulo A
Credenciais Técnicas
Sem isso o sistema literalmente não roda
  • OpenAI API Key (sk-proj-...)
  • Meta System User Token (não expira — via Meta Business Suite)
  • Instagram Account ID (numérico)
  • Facebook Page ID (numérico)
  • ImgBB API Key (gratuito)
  • ElevenLabs API Key + Voice ID se Reels
  • fal.ai API Key se Reels com vídeo IA
  • Notion Token + Database ID se grade editorial
Módulo B
Alma da Marca
O mais importante — define tudo que o sistema vai gerar
  • Nome da marca, handle do Instagram, nicho específico
  • Produto/serviço + como é entregue + palavra do CTA
  • Público: maior dor, maior desejo, o que já tentou, linguagem real
  • O Inimigo: qual sistema falhou + evidência verificável
  • As 5 Praças de Conteúdo (eixos temáticos)
  • Voz da marca: 3 adjetivos + exemplos reais de texto
Módulo C
DNA Visual
Define a estética das imagens geradas
  • 3 contas Instagram com visual admirado
  • Paleta de cores (hex ou descrição)
  • Estilo visual: Gravura / Pintura / Fotorrealista / Cinematográfico
  • Paleta emocional: Violeta / Dourado / Teal / Crimson / Índigo
  • Logotipo/watermark para os slides (arquivo)

🧬 Fase 2 — Transplante de Identidade

Esta é a fase mais crítica. Não é troca de variáveis — é reescrita guiada pelas respostas do onboarding. Aqui é onde a alma do cliente entra no sistema.

Checklist de transplante

1º
Credenciais
  • Preencher .env com todas as credenciais do Módulo A
  • Testar OpenAI: python -c "from openai import OpenAI; print('OK')"
  • Testar Meta API: confirmar resposta 200 com o token
2º
agents/humanizer.md
  • Reescrever "O que a voz É" com os 3 adjetivos do cliente (B6)
  • Reescrever "O que NÃO é" com anti-padrões do nicho específico
  • Criar 2 exemplos de before/after com linguagem REAL do público (B3)
  • Confirmar com cliente: "isso soa como você fala?"
3º
agents/oraculo-revisor.md
  • Critério 3: reescrever inimigo com dados do cliente (B4)
  • Critério 4: reescrever "Comente [PALAVRA] se..." com dor específica
  • Substituir exemplos de espiritualidade por exemplos do nicho
4º
agents/visual-dna.md + canalizador-visual.md
  • Paleta: substituir violeta/teal/ouro pelas cores do cliente (C2)
  • Referências artísticas: ajustar para o estilo do cliente (C1)
  • Tabela cor × estado emocional: reconstruir com paleta do cliente
5º
core/agentes/diretor_artistico.py
  • PRESET_ATMOSPHERE: reescrever para o estilo visual do cliente
  • SLIDE_ENERGY: recalibrar energia por estado emocional no nicho
  • Watermark: mudar para @conta_do_cliente
6º
CLAUDE.md + planner.py
  • CLAUDE.md: atualizar nicho, produto, CTA, praças, handle
  • planner.py: substituir PRACAS pelos 5 pilares do cliente (B5)
  • copywriter/diretor de cena: reescrever SYSTEM_PROMPT com nicho + tom

🎯 Fase 3 — Calibração

Gerar o primeiro carrossel real e validar com o cliente antes de escalar. Nunca pular esta etapa.

Escolher tema de calibração

Relacionado à Praça mais forte · Conectado à dor principal · Sem polêmica · Fácil de avaliar

Gerar carrossel completo

10 slides · Revisar copy e imagens antes de enviar

Enviar para aprovação com roteiro de feedback

"O texto soa como você?" · "A imagem combina?" · "O CTA te faria comentar?"

Ajustar com base no feedback

Máximo 2 rodadas. Se precisar de 3, agendar call de 30min.

Mapa de feedback → ajuste

Feedback do clienteOnde ajustar
"Texto não soa como eu"agents/humanizer.md
"Imagem muito genérica"agents/visual-dna.md + diretor_artistico.py
"Paleta não combina"agents/canalizador-visual.md + preset
"CTA fraco"agents/oraculo-revisor.md Critério 4
"Arco não convenceu"agents/oraculo-revisor.md Critério 2

🔧 Fase 4 — Infraestrutura Técnica

Setup do ambiente

  • Python 3.10+ instalado
  • Node.js 18+ instalado
  • pip install -r requirements.txt
  • npm install (na pasta dashboard/)
  • .env preenchido e testado

Teste de APIs

  • OpenAI: python scripts/test_openai.py
  • Meta: python scripts/test_meta.py
  • ImgBB: python scripts/test_imgbb.py
  • ElevenLabs (se Reels): python scripts/test_elevenlabs.py
  • fal.ai (se Reels): python scripts/test_falai.py

Teste de publicação completo

  • Gerar 1 carrossel completo (10 slides)
  • Confirmar slides salvos em Desktop/carrossel-[slug]/
  • Abrir dashboard → carrossel aparece na lista
  • Clicar "INSTAGRAM" → publicação inicia sem erro
  • ImgBB recebeu as imagens (verificar painel ImgBB)
  • Meta API criou os containers (sem erro 400)
  • Post visível no perfil do Instagram

Erros comuns na publicação

ErroCausaSolução
Meta API 400Token sem permissõesRegenerar System User Token com permissões corretas
Erro ImgBBChave errada ou imagem > 32MBVerificar chave; reduzir qualidade para 85
FINISHED not reachedMeta API demorouAumentar timeout de polling para 10min
Post não apareceConta pessoalConverter para conta profissional no Instagram

📦 Fase 5 — Volume Inicial

Entregar o cliente pronto para operar na primeira semana sem precisar gerar do zero.

O que precisa estar pronto na entrega

  • 5 carrosséis prontos — 1 por Praça (temas definidos com o cliente)
  • Grade editorial da semana 1 no Notion (se incluso)
  • 1 Reel completo (7 cenas) se escopo incluir Reels
  • Arquivos organizados em campanhas/reels/[nome]/cena_XX/

Rodando o planejador editorial

# Ver grade sem criar no Notion
python core/agentes/planner.py --dry-run

# Criar no Notion
python core/agentes/planner.py --dias 30

🚨 Bloqueadores Comuns e Soluções

BloqueadorCausa provávelSolução
Token Meta inválido ou expira Gerou token de usuário comum (expira em 60 dias) em vez de System User Token Guiar via call de 30min no Meta Business Suite — criar Usuário do Sistema
Conta Instagram não é profissional Conta pessoal sem API de conteúdo habilitada Settings → Account → Switch to Professional Account
ImgBB retorna erro Imagens muito grandes (>32MB) ou chave errada Reduzir quality=85 no compose_util · Verificar chave
fal.ai sem créditos no meio do pipeline Saldo zerou durante geração de Reel Verificar saldo antes de iniciar · Cada vídeo 5s ≈ $0.28–0.35
Conteúdo gerado é genérico Onboarding preenchido de forma superficial Revisitar B3 (público) e B6 (voz) — o problema quase sempre está na dor mal identificada
Cliente aprova mas o conteúdo não engaja Identidade mal transplantada ou Praças erradas Analisar 3 posts mais virais do cliente e recalibrar humanizer + revisor
Dashboard não abre Caminho relativo errado no server.js Verificar working directory ao iniciar · Usar caminho absoluto no runtimeArgs
Regra de ouro: 80% dos problemas pós-entrega têm origem no Módulo B do onboarding — especialmente B3 (público) e B4 (inimigo). Se o cliente preencheu de forma genérica, o sistema vai gerar conteúdo genérico. A solução não é técnica — é aprofundar o onboarding.
